**Web AI IDE** is an open source AI coding agent. It runs in your terminal, browser, or desktop — reading, writing, editing, and exploring your codebase with full context. MIT licensed, provider-agnostic, built by Neovim users.

## Usage

Navigate to a project and run Web AI IDE.

```bash
cd /path/to/project
opencode
```

Run `/init` to analyze your project and create an `AGENTS.md` file.

```txt frame="none"
/init
```

:::tip
Commit your project's `AGENTS.md` file to Git. It helps the agent understand your project structure.
:::

---

### Ask questions

Reference files with `@` to give the agent context.

```txt frame="none"
How is authentication handled in @packages/api/src/auth.ts
```

:::tip
Use the `@` key to fuzzy search for files in the project.
:::

---

### Add features

1. Switch to _Plan mode_ with `Tab` — the agent proposes how, doesn't make changes.

2. Describe what you want with enough detail.

3. Iterate on the plan. Give feedback, add screenshots.

4. Switch back to _Build mode_ (`Tab`) and tell it to proceed.

   ```txt frame="none"
   Sounds good! Go ahead and make the changes.
   ```

---

### Undo changes

```txt frame="none"
/undo
```

:::tip
Run `/undo` multiple times to undo multiple changes. Use `/redo` to redo.
:::

## Interfaces

Web AI IDE supports multiple interfaces connecting to the same backend.

### TUI (Terminal)

```bash
opencode                          # Start in current directory
opencode /path/to/project         # Start in specific directory
opencode attach http://host:4096  # Attach to remote server
```

Built with SolidJS rendered to the terminal via [opentui](https://github.com/sst/opentui).

### Web GUI

```bash
opencode web                      # Server (4096) + Web client (3000)
```

The Web GUI provides the same AI chat experience as the TUI. The desktop apps (Tauri and Electron) wrap this interface natively.

### Headless Server

```bash
opencode serve --port 4096
```

Set `OPENCODE_SERVER_PASSWORD` for authentication in production.

## Agents

Two built-in agents, switchable with `Tab`.

### Build (Default)

Full-access agent. Reads, writes, edits files and runs shell commands. Use for development work.

### Plan

Read-only agent for analysis and exploration.
- Denies file edits by default
- Asks permission before running bash commands
- Ideal for exploring unfamiliar codebases or planning changes

### General (Subagent)

Invoked via `@general` in messages. Handles complex multi-step search and exploration tasks internally.

## IDE Integration

### Supported IDEs

VS Code, Cursor, Windsurf, VSCodium, VS Code Insiders

### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Cmd+Esc` / `Ctrl+Esc` | Open in split terminal, or focus existing session |
| `Cmd+Shift+Esc` / `Ctrl+Shift+Esc` | Start a new session |
| `Cmd+Option+K` / `Alt+Ctrl+K` | Insert file reference at cursor |

### File References

```
@src/utils/helper.ts          # Entire file
@src/utils/helper.ts#L42      # Line 42
@src/utils/helper.ts#L37-42   # Lines 37–42
```

The current editor selection is automatically shared with the agent.

### Manual Install

Search **OpenCode** in the extension marketplace, or verify the CLI:

```bash
code --version        # VS Code
cursor --version      # Cursor
windsurf --version    # Windsurf
codium --version      # VSCodium
```

### External Editor

```bash
export EDITOR="code --wait"   # VS Code
export EDITOR="vim"           # Vim / Neovim
```

### Zed

No extension needed. Selections are read directly from Zed's SQLite database.

## Skills System

Agent skills define reusable behavior via `SKILL.md` files — compatible with Claude Code skills.

---

### Place files

Create a folder per skill with a `SKILL.md` inside. Web AI IDE searches:

| Location | Scope |
|----------|-------|
| `.opencode/skills/<name>/SKILL.md` | Project config |
| `~/.config/opencode/skills/<name>/SKILL.md` | Global config |
| `.claude/skills/<name>/SKILL.md` | Project, Claude-compatible |
| `~/.claude/skills/<name>/SKILL.md` | Global, Claude-compatible |
| `.agents/skills/<name>/SKILL.md` | Project, agent-compatible |
| `~/.agents/skills/<name>/SKILL.md` | Global, agent-compatible |

For project-local paths, Web AI IDE walks up from the current working directory until it reaches the git worktree.

---

### Write frontmatter

Each `SKILL.md` must start with YAML frontmatter.

```yaml
---
name: my-skill
description: What this skill does and when to use it
---
```

`name` must be 1–64 characters, lowercase alphanumeric with single hyphen separators, and match the directory name containing `SKILL.md`.

---

### Example

Create `.opencode/skills/git-release/SKILL.md`:

```yaml
---
name: git-release
description: Create consistent releases and changelogs
---

## What I do
- Draft release notes from merged PRs
- Propose a version bump
- Provide a copy-pasteable `gh release create` command

## When to use me
Use this when you are preparing a tagged release.
Ask clarifying questions if the target versioning scheme is unclear.
```

---

### Remote skills

Add a URL to `config.skills.urls` that serves an `index.json`:

```json
{
  "skills": [
    { "name": "my-skill", "files": ["SKILL.md", "script.py"] }
  ]
}
```

Skills are cached in `~/.cache/opencode/skills/`.

---

### Permissions

Control which skills agents can access with pattern-based permissions:

```json
{
  "permission": {
    "skill": {
      "*": "allow",
      "internal-*": "deny",
      "experimental-*": "ask"
    }
  }
}
```

| Permission | Behavior |
|------------|----------|
| `allow` | Skill loads immediately |
| `deny` | Skill hidden from agent |
| `ask` | User prompted before loading |

### Custom themes

Create a JSON file in one of the theme directories.

```bash
mkdir -p ~/.config/opencode/themes    # User-wide
mkdir -p .opencode/themes              # Project-specific
```

```json title="my-theme.json"
{
  "$schema": "https://opencode.ai/theme.json",
  "theme": {
    "primary": { "dark": "#88C0D0", "light": "#5E81AC" },
```

## Architecture

```
┌────────────────────────────────────────────┐
│                 Client Layer                 │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐ │
│  │   TUI    │  │ Web GUI  │  │ Desktop  │ │
│  │ Terminal │  │ Browser  │  │ Tauri/El │ │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘ │
│       └──────────────┼──────────────┘      │
│               ┌──────┴──────┐               │
│               │  TypeScript  │              │
│               │     SDK      │              │
│               └──────┬──────┘               │
├──────────────────────┼──────────────────────┤
│                      │     Server Layer      │
│  ┌───────────────────┴────────────────────┐ │
│  │        Hono HTTP Server (Effect)       │ │
│  ├──────────┬──────────┬────────┬────────┤ │
│  │  Agent   │  Config  │ SQLite │  Auth  │ │
│  │ Runtime  │  Manager │  (ORM) │ (OAuth)│ │
│  └──────────┴──────────┴────────┴────────┘ │
└────────────────────────────────────────────┘
```

### Tech Stack

| Layer | Technology |
|-------|-----------|
| Runtime | Bun, TypeScript 5.8 |
| Server | Hono 4, Effect 4.0 |
| Database | SQLite (Drizzle ORM) |
| Validation | Zod 4 |
| Frontend | SolidJS 1.9, Vite 7, TailwindCSS 4 |
| Desktop | Tauri v2, Electron |
| AI SDK | Vercel AI SDK (`ai` 6.0) |

### Design Principles

- **Provider-agnostic**: Anthropic, OpenAI, Google, and local models
- **Terminal-first**: Built by Neovim users — pushing terminal UX boundaries
- **Client/Server**: TUI is one client; drive the server remotely
- **LSP-native**: Built-in Language Server Protocol support

## Development

### Prerequisites

- Bun >= 1.3
- (Optional) Rust toolchain for Tauri desktop

### Setup

```bash
bun install
bun dev                       # Start TUI dev mode
bun dev <directory>           # Start in specific directory
```

### Common Commands

```bash
bun typecheck                                 # Type checking
bun lint                                      # Linting
cd packages/opencode && bun test              # Run tests
./packages/opencode/script/build.ts --single   # Standalone binary
./packages/sdk/js/script/build.ts              # Regenerate SDK
```

### Debugging

```bash
# Debug server
bun run --inspect=ws://localhost:6499/ --cwd packages/opencode ./src/index.ts serve --port 4096

# Debug TUI
bun run --inspect=ws://localhost:6499/ --cwd packages/opencode --conditions=browser ./src/index.ts
```
