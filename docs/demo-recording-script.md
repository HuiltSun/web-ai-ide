# Code Neural Map — 演示录制脚本

**作者**: 孙以恒、魏征征  
**功能**: Code Neural Map 神经星图 IDE 模式  
**建议时长**: 3–5 分钟

---

## 准备工作（录制前）

### 1. 启动后端（终端 1）

```bash
cd E:\web-ai-ide
bun dev
```

等待看到类似 `Server listening on http://localhost:...` 的输出。

### 2. 启动前端（终端 2）

```bash
cd E:\web-ai-ide\packages\app
bun dev
```

前端默认运行在 `http://localhost:3000`。

### 3. 打开浏览器

访问 `http://localhost:3000`，正常进入 opencode 主界面。

### 4. 获取项目路径的 URL 编码

在浏览器控制台（F12）执行：

```javascript
// 获取当前项目目录的 base64 编码
btoa("E:\\web-ai-ide")
// 或者从地址栏的现有 URL 中直接复制 /:dir 部分
```

假设编码结果为 `RTpcd2ViLWFpLWlkZQ==`，则神经图地址为：

```
http://localhost:3000/RTpcd2ViLWFpLWlkZQ==/neural-map
```

> **提示**: 正常进入项目后，地址栏会是 `http://localhost:3000/<base64dir>`，把后面改为 `/neural-map` 即可。

## 录制脚本（分镜）

### 第 1 幕：引入（0:00–0:30）

**画面**: 展示普通的 opencode 主界面（代码编辑 / 对话界面）

**旁白**:
> "这是我们基于 opencode 构建的 Web AI IDE。传统 IDE 里，理解陌生代码库要靠手动翻文件、打命令。今天我们给它加了一个全新的模式——Code Neural Map 代码神经星图，让 AI 带你用完全不同的方式探索代码。"

**操作**: 鼠标悬停在地址栏，把 URL 末尾改为 `/neural-map`，按回车。

---

### 第 2 幕：神经图加载（0:30–1:00）

**画面**: 页面出现"正在构建代码神经图..."加载状态 → 图谱渲染完成

**要截图的关键画面 📸**:
- 加载动画（可选）
- **图谱首次出现的全景图** ← 最重要截图

**旁白**:
> "进入模式后，系统自动解析项目的所有 import 关系，把代码库渲染成一张有机的神经网络图。每个节点是一个代码模块，曲线是依赖关系，光晕大小代表模块的活跃度。"

**操作**: 用鼠标滚轮缩放、拖拽平移图谱，展示全局视图。

---

### 第 3 幕：点击节点（1:00–2:00）

**画面**: 点击一个高亮节点（推荐点击 `agent` 或 `session` 模块）

**要截图的关键画面 📸**:
- 节点被选中、右侧面板出现的状态

**旁白**:
> "点击任意节点，右侧面板立即显示模块元信息：文件数、代码行数、活跃度评分。"

**操作**: 点击"🤖 开始解读这个模块"按钮。

**画面**: AI 向导加载中 → 介绍 + 问题出现

**要截图的关键画面 📸**:
- **AI 向导介绍 + 猜一猜题目同框** ← 核心截图

**旁白**:
> "AI 不等你提问——它主动介绍这个模块是什么、做什么，然后出一道 Socratic 问题来测试你的理解。这就是我们打破的第三个范式：不是用户问 AI，而是 AI 主导学习节奏。"

---

### 第 4 幕：回答问题（2:00–2:45）

**操作**: 在输入框中输入一个答案（随便输入，比如"负责处理用户会话"），按回车或点击 ↵。

**画面**: AI 反馈出现（绿色 ✓ 框）

**要截图的关键画面 📸**:
- 答案 + AI 反馈 + 推荐下一站同框

**旁白**:
> "AI 根据你的回答给出即时反馈，然后推荐下一个逻辑上相关的模块。学习路径不是固定的——它根据你的理解情况动态调整。"

---

### 第 5 幕：标记已懂（2:45–3:15）

**操作**: 点击"标记已懂"按钮。

**画面**:
- 节点颜色变绿，出现 ✓ 徽章
- 顶部进度条数字增加

**要截图的关键画面 📸**:
- **顶部进度条 + 绿色已理解节点 + 推荐路径同框** ← 展示进度追踪

**旁白**:
> "标记已懂后，节点变绿，顶部进度条实时更新。这个进度是持久化的——关掉页面重新进来，已理解的节点仍然保留。"

---

### 第 6 幕：导航下一站（3:15–3:45）

**操作**: 点击"→ 下一站"按钮，自动跳转到 AI 推荐的下一个模块并立即解读。

**旁白**:
> "点击下一站，AI 自动带你到它认为最应该接下来理解的模块，并立即开始新一轮解读。你不需要决定看什么——就像有一个资深同事在旁边带你过代码。"

---

### 第 7 幕：退出模式（3:45–4:00）

**操作**: 点击右上角"✕ 退出模式"。

**画面**: 返回普通 opencode 界面。

**旁白**:
> "一键退出，回到正常的 IDE 模式。神经图模式和普通编码流程完全独立，不会影响现有功能。"

## 录屏建议

- **分辨率**: 1280×800 或更高，建议全屏浏览器
- **工具**: OBS Studio（免费）或 剪映桌面版
- **窗口**: 只录浏览器窗口，隐藏终端
- **鼠标**: 移动慢一点，让观众跟上节奏
- **旁白**: 可以先录画面，后期配音效果更好
