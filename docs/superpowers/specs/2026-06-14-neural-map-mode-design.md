## 概述

**Code Neural Map** 是一种全新的 IDE 模式，用户可从普通代码编辑界面一键切换进入。它将代码库渲染为有机神经网络图，由 AI 向导主导探索节奏，帮助用户在不读文件、不打命令的情况下深度理解陌生代码库。

核心理念：**打破 IDE 的三重范式**

| 范式维度 | 传统 IDE | Code Neural Map |
|---|---|---|
| 输入 | 打字输入代码/命令 | 点击节点、回答 AI 问题、拖拽探索 |
| 输出 | 文字/代码块 | 神经图谱 + 活跃度脉冲 + AI 气泡 + 进度可视化 |
| 节奏 | 用户主动提问，AI 被动响应 | AI 主动引导，设计学习路径，出题测验 |

---

## 目标用户场景

**主场景**：开发者拿到一个陌生代码库，需要快速理解其结构和核心模块关系。

**次场景**：开发者边探索边学习，AI 像导师一样测试理解程度，记录已掌握节点。

**主画布**（左侧，~75% 宽度）：
- SVG 渲染的力导向图，有机曲线连接

- 节点点击后高亮并在右侧面板展示详情
- AI 推荐路径以虚线箭头叠加显示

**AI 向导面板**（右侧，~25% 宽度）：
- 顶部：当前选中节点元信息（路径、行数、标签）
- 中部：AI 对话流（AI 主动说明 → 出题 → 用户答 → AI 反馈）
- 底部：操作按钮 + AI 推荐下一探索路径

**顶部栏**：
- 模式标签（可点击退出）
- 全局知识进度条（已理解节点 / 总节点数）

## 核心功能

### 1. 代码图谱构建

**数据来源**：
- **主要**：静态解析 `import`/`require` 语句建立有向图（无需 LSP，实现简单、准确率 ~85%）
- **增强**：若 LSP 已启动，额外获取类型引用关系，补充动态依赖
- 读取 `packages/opencode/src/` 目录结构作为节点基础数据

**节点属性**：
```typescript
interface GraphNode {
  id: string           // 模块路径
  label: string        // 显示名称
  file_count: number   // 文件数量
  line_count: number   // 代码行数
  dependencies: string[]  // 依赖的节点 ID
  dependents: string[]    // 依赖此节点的 ID
  activity: number     // 活跃度评分（基于 git 提交频率）
  understood: boolean  // 用户是否已标记为理解
}
```

**布局算法**：力导向图（D3-force 或自实现），核心模块居中，外围模块按依赖层级分布。

### 2. 神经图视觉渲染

**技术方案**：SolidJS + SVG（节点数 < 200 时性能可接受；超出则切换 Canvas）

**视觉元素**：

| 元素 | 实现 |
|---|---|
| 节点光晕 | `<circle>` + SVG `feGaussianBlur` filter，半径随活跃度变化 |
| 有机曲线连接 | SVG `<path>` 贝塞尔曲线，`Q` 控制点随机偏移产生有机感 |
| 流动粒子 | CSS `@keyframes` 或 SVG `animateMotion`，沿路径运动 |
| 脉冲动画 | 同心圆 opacity 从 0.8 → 0，`animation-delay` 错开 |
| 节点状态徽章 | 绝对定位的 SVG `<text>` 或 `<circle>` |

**节点颜色语义**：
- 蓝色 (#58a6ff)：核心/被高度依赖的模块
- 绿色 (#3fb950)：已理解的节点
- 黄色 (#d29922)：高活跃度/复杂度节点
- 紫色 (#bc8cff)：数据层模块
- 红色 (#f78166)：网关/IO 模块
- 灰色 (#8b949e)：未探索节点

### 3. AI 向导系统

**AI 主动行为**（不需要用户提问）：
1. 用户进入模式时，AI 自动介绍整体架构（一段话）
2. 用户点击节点时，AI 主动给出节点介绍 + 一道猜测题
3. 用户标记"已懂"后，AI 推荐下一个逻辑上相关的节点

**AI 对话格式（固定结构）**：
```
[介绍] 这个模块负责...（2-3句话）
[猜题] 🎯 你觉得...会做什么？（单选或开放题）
[反馈] ✓ 对！/ 稍有偏差，实际上...（基于用户答案）
[引导] → 建议接下来看 X，因为它直接依赖这里的 Y
```

**Prompt 设计原则**：
- 系统 prompt 注入代码图谱的结构数据（节点元信息）
- 用户当前选中节点的文件摘要（前 50 行 + 导出符号列表）
- 已理解节点列表（AI 避免重复介绍）
- 要求 AI 输出结构化 JSON，前端解析后渲染成对话气泡

### 4. 知识追踪系统

**存储**：复用现有 SQLite 存储层，新增 `neural_map_progress` 表：
```sql
CREATE TABLE neural_map_progress (
  session_id TEXT,
  node_id TEXT,
  understood_at INTEGER,  -- Unix timestamp，NULL 表示未理解
  notes TEXT,             -- 用户标注的笔记
  PRIMARY KEY (session_id, node_id)
);
```

**进度可视化**：
- 顶部进度条：已理解节点数 / 总节点数
- 已理解节点：绿色 ✓ 徽章 + 降低光晕强度
- 全局"知识热力图"：可选的覆盖层，展示探索密度

### 模式入口


### 数据流

```
代码库文件系统
    ↓ (LSP / import 解析)
GraphBuilder (新增服务，packages/opencode/src/neural-map/graph.ts)
    ↓ (HTTP API: GET /neural-map/graph)
前端 GraphStore (SolidJS signal)
    ↓ (渲染)
SVG Neural Map Canvas
    ↑ (节点点击事件)
AI Guide Service (packages/opencode/src/neural-map/guide.ts)
    ↓ (调用现有 AI SDK)
AI 向导面板
```

### 新增文件清单

```
packages/opencode/src/neural-map/
├── graph.ts          # 代码图谱构建：解析依赖、计算活跃度、输出节点/边数据
├── guide.ts          # AI 向导：prompt 构建、结构化输出解析、对话状态管理
└── storage.ts        # 知识进度读写（SQLite）

packages/app/src/modes/neural-map/
├── index.tsx         # 模式入口路由
├── Canvas.tsx        # SVG 神经图主画布组件

```

## 不在范围内（本期不做）

- 移动端 / Tauri 客户端支持（仅 Electron）
- 3D 渲染（保持 2D SVG）
- 多用户协同探索
- 代码编辑功能（此模式为只读探索）
- 超过 500 节点的超大型代码库优化

---

## 成功标准

1. 用户进入模式后 10 秒内看到完整的代码图谱
2. 点击任意节点后 3 秒内 AI 向导给出介绍和题目
3. 「已懂」标记持久化，重新进入模式后保留进度
4. 图谱在 1280×800 分辨率下无需横向滚动即可看到主要节点

---

## 风险与缓解

| 风险 | 缓解方案 |
|---|---|
| LSP 未启动时增强功能缺失 | 静态分析为主路径，LSP 缺失仅影响类型引用关系，核心图谱不受影响 |
| SVG 节点过多性能下降 | 节点 > 100 时自动合并同目录节点为"文件夹节点" |
| AI prompt 过长 | 每次只传当前节点 + 直接邻居的摘要，不传全图 |
| 与现有路由冲突 | 新增独立路由 `/neural-map`，不修改现有页面逻辑 |
