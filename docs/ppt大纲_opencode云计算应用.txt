第 2 章  背景：云时代的开发者痛点
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第 3 页】传统本地开发的局限
  - 环境配置繁琐：不同机器、不同系统难以统一
  - 资源受限：大型编译/推理任务超出本地算力
  - 协作困难：代码、上下文难以实时共享
  - 迁移成本高：换机器 = 重建开发环境
  ▶ 引出问题：能否把开发环境搬到云上？

【第 4 页】AI 编程助手的新要求
  - 大语言模型推理需要调用云端 API（Anthropic / OpenAI / 本地）
  - 上下文窗口大，需要低延迟的网络通道
  - 工具调用（文件读写、终端执行）需要紧密集成本地或远程文件系统
  - 多会话并发、历史持久化 → 天然适合 B/S 架构
  ▶ 引出方向：Web IDE + 云后端 = 完整解决方案

【第 5 页】项目定位
  - 开源 AI Coding Agent（MIT 协议）
  - 三种运行形态：
      ① CLI / TUI：终端交互，适合 SSH 场景
      ② Web GUI：浏览器访问，零安装，跨平台
      ③ 桌面应用：Electron 封装 Web UI，离线可用
  - 核心理念：同一套后端服务 → 多前端消费

【第 8 页】前后端分离的云原生特征
  - 前端：纯静态资源，可部署到 CDN（Cloudflare / OSS）
  - 后端：无状态 API + 有状态 PTY Session，可容器化
  - 通信：标准 HTTP + WebSocket，适配反向代理（Nginx / Caddy）
  - 认证：Basic Auth 可接入 OAuth / SSO
  - 持久化：SQLite 可迁移至 PostgreSQL（云数据库）

【第 9 页】WebSocket 事件总线设计
  - 终端输出实时推送：/pty/{id}/connect（二进制流）
  - 会话状态变更：EventBus 广播到所有订阅者
  - 文件变更监听：inotify / FSEvents → Web 前端实时同步
  - 断线重连机制：指数退避，状态无缝恢复
  ▶ 支撑多用户同时接入同一工作空间

第 5 章  核心功能模块
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第 10 页】AI 对话会话（Session）
  - 每个会话独立管理上下文、消息历史
  - 支持子会话（parentID）：Agent 嵌套调用
  - 会话持久化到 SQLite，跨浏览器恢复
  - Worktree 支持：同一项目多分支并行开发
  - 支持 Follow-up 消息队列（异步 AI 响应期间排队）

【第 11 页】嵌入式代码编辑器
  - 基于 CodeMirror 6，纯 Web 技术栈
  - 支持语言：JS/TS/Python/Go/Rust/SQL/HTML/CSS 等
  - 主题系统：与应用整体配色同步（CSS 变量驱动）
  - AI Diff 预览：接受/拒绝模型建议的代码修改
  - 文件变更实时感知（磁盘变更提示重载）

【第 12 页】嵌入式终端
  - 基于 Ghostty（WebAssembly 版本）渲染
  - 后端通过 node-pty 管理 PTY 进程
  - WebSocket 实时双向流：键入 → PTY → 输出
  - 多终端标签，支持拖拽排序
  - 断线缓冲恢复：重连后补发离线期间输出

【第 13 页】文件树与变更审查
  - 目录树递归浏览，支持双击在编辑器打开
  - Diff 视图：会话级变更 / Git 未提交 / 分支差异
  - 行级评论：在 Diff 中标注后作为 AI 上下文
  - 评审面板：split / unified 两种 diff 视图

【第 14 页】插件与 MCP 扩展
  - Plugin API：生命周期钩子（shell.env / provider 等）
  - MCP（Model Context Protocol）：
      - 标准化 AI ↔ 工具的通信协议
      - 支持注册外部工具供 Agent 调用
      - 可接入数据库查询、API 调用、文档检索等
  - 开放扩展点，适合企业定制集成

第 6 章  云部署方案
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第 15 页】方案一：本地 + 远程访问
  ┌─────────────────────────────────────┐
  │  开发者本机 / 云服务器               │
  │  WEB AI IDE web → localhost:4096      │
  │  Nginx 反向代理 + HTTPS             │
  └─────────────────────────────────────┘
  - 成本极低，适合个人开发者
  - frp / Cloudflare Tunnel 实现公网访问
  - 手机/平板浏览器即可操作

【第 16 页】方案二：云服务器部署
  - 云主机（ECS / EC2）运行 WEB AI IDE serve
  - Docker 容器化：
      FROM node:20-alpine
      RUN npm i -g WEB AI IDE-ai
      EXPOSE 4096
      CMD ["WEB AI IDE", "web", "--port", "4096"]
  - 通过 Nginx 提供 HTTPS + 身份认证
  - 工作目录挂载云盘（NAS / OSS FUSE）

【第 17 页】方案三：Kubernetes 集群部署
  - 每个用户 / 项目独立 Pod（隔离性强）
  - Persistent Volume 挂载代码目录
  - Service + Ingress 路由到各用户实例
  - HPA 水平扩展：按活跃会话数自动伸缩
  - 适合企业级多租户场景

【第 18 页】方案四：Serverless + 边缘计算
  - 静态前端：Cloudflare Pages / Vercel 全球 CDN
  - API 层：Cloudflare Workers / AWS Lambda（轻量请求）
  - PTY / AI 会话：长连接服务，仍需常驻进程（EC2 / ECS）
  - 混合架构：静态资源边缘化 + 有状态服务中心化
  ▶ 兼顾响应速度与成本

【第 19 页】安全性设计
  - 传输层：TLS 1.3 全程加密
  - 认证层：Basic Auth / Token，可接入 OAuth 2.0
  - 隔离层：
      - Docker namespace 隔离进程
      - 只读文件系统挂载（只开放项目目录）
      - PTY 会话权限控制
  - 审计：会话操作全量记录到 SQLite

第 7 章  AI 能力接入与云 API
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第 20 页】多模型支持架构
  ┌────────────────────────────────────────────┐
  │            WEB AI IDE Agent 层                │
  │  build mode（全权限）/ plan mode（只读）     │
  └──────────────┬─────────────────────────────┘
                 │ Provider 抽象层
  ┌──────────────▼─────────────────────────────┐
  │  Anthropic Claude   OpenAI GPT-4o          │
  │  Google Gemini      AWS Bedrock            │
  │  Ollama（本地）     OpenRouter             │
  └────────────────────────────────────────────┘
  - 统一接口，切换模型无需改代码
  - 支持 Prompt Caching（降低重复上下文成本）

【第 21 页】AI 工具调用链（Tool Use）
  用户输入 → Agent 决策 → 工具调用：
    · file.read / file.write   读写文件
    · bash.execute             运行终端命令
    · web.search               联网搜索（MCP）
    · git.diff / git.commit    版本控制
    · lsp.hover / lsp.complete 语言服务
  → 结果反馈给模型 → 下一步推理
  ▶ 形成完整的「感知 → 规划 → 执行」循环

【第 22 页】上下文管理与 Token 优化
  - 滑动窗口：超出上下文限制时自动裁剪历史
  - Prompt Caching：相同前缀缓存复用，节省 API 费用
  - 会话摘要：长对话自动压缩为摘要继续
  - 文件内容按需加载（不一次性塞满上下文）
  ▶ 在云 API 成本与效果之间取得平衡

第 8 章  应用场景
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第 23 页】场景一：远程开发环境
  问题：笔记本算力不足，项目依赖复杂
  方案：
    - 云服务器 8C32G 运行 WEB AI IDE + 代码库
    - iPad / 轻薄本浏览器接入，随时随地编码
    - AI Agent 自动处理依赖安装、测试运行
  效果：本地设备零配置，算力按需付费

【第 24 页】场景二：教学 / 实验室环境
  问题：学生机器性能参差不齐，环境配置耗时
  方案：
    - K8s 集群为每位学生分配独立 Pod
    - 浏览器直接访问，环境完全一致
    - AI Agent 辅助解答报错、代码审查
    - 教师实时查看学生 session，远程指导
  效果：环境统一，AI 辅助学习，运维成本低

【第 25 页】场景三：CI/CD 中的 AI Review
  问题：代码评审人力成本高，覆盖率低
  方案：
    - WEB AI IDE 部署为云服务，暴露 REST API
    - PR 触发 → WEB AI IDE Agent 自动拉取 diff
    - plan mode（只读）分析代码质量、安全漏洞
    - 将 review 结果推送到 GitHub PR 评论
  效果：AI 先行 review，人工聚焦高风险改动

【第 26 页】场景四：企业内部知识库编程助手
  问题：通用 AI 模型不了解内部代码规范、业务逻辑
  方案：
    - 通过 MCP 接入内部文档、API 文档、设计规范
    - Plugin 注入公司专属 prompt 前缀
    - 私有化部署，代码不出内网
    - 多项目统一管理，按团队隔离
  效果：AI 具备公司上下文，输出符合内部标准

【第 28 页】未来展望
